데이터 수집 및 전처리 계획

노인 정신건강 모니터링을 위한 데이터 확보 및 처리 전략

1. 데이터 수집 개요

1.1 필요 데이터 유형

  • 노인 음성/대화 데이터: 전화 통화 녹음, 일상 대화 녹음
  • 정신건강 평가 데이터: 인지기능, 우울증, 사회적 고립도 관련 평가 지표
  • 시계열 데이터: 동일 대상자의 시간 경과에 따른 대화 및 정신건강 변화 데이터

1.2 데이터 수집 방법

AI-Hub 데이터셋 활용

  • AI-Hub에서 제공하는 노인 자유대화 음성 데이터 활용
  • 60세 이상 남녀의 실제 일상 언어 특성을 반영한 데이터셋
  • 오디오(음성 파일), 텍스트(전사 결과), 발화자 정보 등 포함
  • 비문법적 표현, 말 줄임, 반복 등 실제 노인 언어의 특징 포함

자체 데이터 수집

  • 학부생들의 가족(조부모) 대상 전화 대화 수집
  • 노인복지센터 협력을 통한 참여자 모집 및 데이터 수집
  • 기존 정신건강 평가 도구를 활용한 정신건강 상태 평가 데이터 수집

1.3 데이터 수집 규모

  • AI-Hub 노인 대화 데이터: 최소 100시간 분량
  • 자체 수집 데이터: 약 20명의 노인 대상, 각 5-10회 정도의 전화 통화 데이터(총 100-200회 통화)
  • 정신건강 평가 데이터: 동일 참여자 대상 최소 3회 이상의 주기적 평가 데이터

2. 데이터 수집 윤리 및 법적 고려사항

2.1 개인정보 보호

  • 모든 참여자로부터 서면 동의서 취득
  • 개인식별정보 제거 및 익명화 처리
  • 개인정보보호법 및 생명윤리법 준수

2.2 데이터 보안

  • 수집된 데이터는 암호화하여 안전하게 보관
  • 접근 권한이 있는 연구자만 데이터 접근 가능
  • 연구 목적 외 데이터 사용 금지

개인정보 보호 원칙

본 프로젝트는 취약계층인 노인을 대상으로 하는 데이터 수집을 포함하기 때문에 엄격한 윤리적 기준과 개인정보 보호 원칙을 준수합니다. 모든 참여자에게 연구 목적과 방법을 이해하기 쉽게 설명하고, 자발적 참여와 중도 철회 권리를 보장합니다.

3. 데이터 전처리 방법

3.1 음성 데이터 전처리

오디오 파일 정제

  • 노이즈 제거 및 음성 향상
  • 음성 파일 포맷 및 샘플링 레이트 표준화
  • 음성 파일 분할 및 라벨링

음성-텍스트 변환(STT)

  • 한국어 STT 모델을 활용한 음성의 텍스트 변환
  • 노인 발화 특성을 고려한 STT 모델 선택 또는 조정
  • 전사 결과 수동 검증 및 보정

3.2 텍스트 데이터 전처리

텍스트 정제

  • 특수문자, 불필요한 기호 제거
  • 맞춤법 및 오타 검사 (단, 노인 발화의 특성은 보존)
  • 텍스트 정규화 및 표준화

텍스트 분석 준비

  • 형태소 분석 및 품사 태깅
  • 문장 분할 및 대화 턴 식별
  • 감정 및 주제 관련 키워드 태깅

4. 데이터셋 구성 및 관리

4.1 데이터셋 구조

기본 구조

  • 원시 데이터 (Raw Data): 원본 음성 파일, 원본 평가 데이터
  • 전처리 데이터 (Processed Data): 정제된 음성, 텍스트, 평가 데이터
  • 분석용 데이터 (Analysis Data): 모델 학습 및 평가용으로 구조화된 데이터

메타데이터 관리

  • 데이터 수집 일시, 방법, 대상자 정보
  • 전처리 과정 및 적용된 방법
  • 데이터 버전 관리

5. 데이터 활용 계획

5.1 모델 학습 및 검증용 데이터 분할

데이터 분할 비율

  • 학습 데이터셋 (Training Dataset): 전체 데이터의 70%
  • 검증 데이터셋 (Validation Dataset): 전체 데이터의 15%
  • 테스트 데이터셋 (Test Dataset): 전체 데이터의 15%

5.2 데이터 활용 방안

LLM 파인튜닝

  • 1차 파인튜닝: 일반 정신건강 도메인 데이터 활용
  • 2차 파인튜닝: 노인 특화 언어 패턴 데이터 활용

시계열 분석 알고리즘 개발

  • 정신건강 변화 추적을 위한 시계열 데이터 활용
  • 패턴 인식 및 이상 징후 감지 알고리즘 개발
