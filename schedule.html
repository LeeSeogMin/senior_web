월별 구현 계획

시니어 전화상담 기반 정신건강 모니터링 시스템의 단계별 개발 일정

1. 프로젝트 일정 개요

본 프로젝트는 2024년 4월부터 9월까지 6개월간 진행되며, 각 단계별 세부 계획은 다음과 같습니다.

프로젝트 기간

  • 시작일: 2024년 4월 1일
  • 종료일: 2024년 9월 30일
  • 총 기간: 6개월

2. 1단계: 데이터 구축 및 모델 개발 기반 마련 (4-5월)

4월

  • 1주차: 프로젝트 킥오프 미팅
  • 2주차: 데이터 수집 계획 수립
  • 3주차: AI-Hub 노인 음성/대화 데이터셋 확보
  • 4주차: 정신건강 평가 데이터 수집 시작

5월

  • 1주차: 데이터 전처리 파이프라인 구축
  • 2주차: 노인 특화 정신건강 평가 지표 설계
  • 3주차: 오픈소스 LLM 선정 및 테스트
  • 4주차: 1차 파인튜닝을 위한 데이터셋 준비

1단계 주요 산출물

  • 데이터 수집 및 전처리 계획서
  • 노인 특화 정신건강 평가 지표 설계서
  • 기반 LLM 모델 선정 보고서
  • 전처리된 학습 데이터셋

3. 2단계: 핵심 시스템 개발 (6-7월)

6월

  • 1주차: LLM 1차 파인튜닝 시작
  • 2주차: 노인 특화 언어 패턴 분석
  • 3주차: 시계열 데이터 분석 알고리즘 설계
  • 4주차: 대화 텍스트 분석 모듈 개발 시작

7월

  • 1주차: LLM 2차 파인튜닝 실시
  • 2주차: 정신건강 상태 평가 모듈 개발
  • 3주차: 변화 감지 알고리즘 구현
  • 4주차: 초기 프로토타입 테스트

LLM 파인튜닝

  • 일반 정신건강 도메인 1차 파인튜닝
  • 노인 특화 언어 패턴 2차 파인튜닝
  • 모델 성능 평가 및 최적화

시계열 분석

  • LSTM 기반 시계열 모델 개발
  • 정신건강 지표 변화 패턴 분석
  • 개인별 기준선 설정 알고리즘

프로토타입 개발

  • 기본 UI/UX 설계
  • 핵심 기능 구현
  • 내부 테스트 실시

4. 3단계: 시스템 통합 및 평가 (8-9월)

8월

  • 1주차: 대시보드 UI/UX 설계
  • 2주차: 시각화 모듈 개발
  • 3주차: 알림 시스템 구현
  • 4주차: 시스템 통합 테스트

9월

  • 1주차: 소규모 테스트 그룹 모집
  • 2주차: 시스템 성능 평가
  • 3주차: 사용자 피드백 수집 및 반영
  • 4주차: 최종 보고서 작성

최종 산출물

  • 노인 정신건강 모니터링 시스템 프로토타입
  • 시스템 사용 매뉴얼
  • 성능 평가 보고서
  • 향후 연구 및 개발 방향 제안서

6. 리스크 관리 계획

잠재적 리스크 및 대응 전략

  • 데이터 수집 지연: 대체 데이터셋 활용 계획 수립, 수집 방법 다변화
  • 모델 성능 미달: 추가 파인튜닝 및 모델 구조 수정, 대체 모델 검토
  • 시스템 통합 문제: 단계적 통합 전략 수립, 모듈화된 설계로 문제 격리
  • 테스트 그룹 모집 어려움: 대안 평가 방안 마련, 인센티브 제공 검토
  • 컴퓨팅 자원 부족: 클라우드 컴퓨팅 자원 활용, 모델 경량화 방안 검토
